각 항목을 PASS/FAIL로 표시하고, FAIL이면 파일:라인과 수정 코드를 함께 제시해.
마지막에 총점과 우선 수정 TOP 3를 알려줘.

## 1. 빌드 및 타입 안전성
- npm run build 실행해서 에러 확인
- npx tsc --noEmit으로 타입 에러 확인
- any 타입이 쓰인 곳 찾기
- props 타입이 누락된 컴포넌트 찾기

## 2. 디자인 시스템 일치 (docs/DESIGN-SYSTEM.md 대조)
- 배경색이 #0C0C0E / #161618 / #1E1E22 외의 값 사용
- 액센트가 #D4A853 외의 포인트 컬러 사용
- 텍스트가 #FAFAFA / #A1A1AA / #71717A 외의 값 사용
- 숫자에 Oswald, 로고에 Rajdhani, 본문에 Noto Sans KR 적용 여부
- 카드 radius 10~14px, 바텀탭 54px 준수 여부

## 3. 코드 구조 및 품질
- 150줄 넘는 컴포넌트 분리 필요 여부
- 3곳 이상 중복되는 패턴 공통화 필요 여부
- 하드코딩된 문자열이나 숫자 (constants.ts로 빼야 할 것)
- console.log 잔존 여부
- useEffect 의존성 배열 누락
- 인라인 객체/함수로 인한 불필요한 re-render 가능성

## 4. 기능 플로우 및 동작 검증
docs/PROGRESS.md 체크리스트에서 완료 표시된 기능들이 실제로 동작하는지 코드를 추적해서 확인해줘.

확인 방식: 각 기능의 진입점(버튼/이벤트) -> 핸들러 -> API 호출/DB 연동 -> 결과 반영까지 코드 경로를 추적.

주요 플로우:
- 프로필 조회: 페이지 로드 -> Supabase fetch -> 프로필 카드 렌더링
- 프로필 편집: 편집 버튼 -> 인라인 편집 -> 저장 -> DB 반영
- 영상 업로드: 파일 선택 -> 용량/길이 체크 -> R2 업로드 -> DB 저장 -> 목록 반영
- 태그/Featured: 태그 지정 -> Top Clip 설정 -> Featured 추가 -> 프로필 반영
- 측정/메달: 기록 입력 -> 메달 조건 체크 -> 자동 부여 -> 레벨 재계산
- 피드: 행동 발생 -> 피드 카드 자동 생성 -> 홈 피드에 노출
- 소셜: 팔로우/응원/댓글 -> DB 저장 -> 카운트 업데이트 -> UI 반영
- 팀: 생성 -> 초대코드 -> 가입 -> 멤버 목록 -> 앨범

끊어진 플로우(핸들러는 있는데 API 미연결, DB 저장 누락 등)를 찾아서 보고해줘.

## 5. 에러 처리 및 UX
- 네트워크 에러 시 사용자에게 보여주는 UI가 있는지
- 데이터 로딩 중 스켈레톤/스피너가 있는지
- 데이터가 비어있을 때 빈 상태(empty state) UI가 있는지
- 업로드/저장 실패 시 재시도 UI가 있는지
- 낙관적 업데이트가 실패했을 때 롤백 처리가 있는지

## 6. 성능 및 반응형
- 이미지/영상에 lazy loading 적용 여부
- 긴 리스트에 페이지네이션 또는 무한스크롤 여부
- 동일 API를 중복 호출하는 곳
- 375px~430px 범위에서 레이아웃 깨질 가능성이 있는 코드
