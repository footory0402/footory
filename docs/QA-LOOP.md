# FOOTORY 반복 QA 루프 (시나리오 + 관점 기반)

> 기준 문서: `docs/SPEC.md`, `docs/PROGRESS.md`  
> 목적: "한 번 테스트"가 아니라, 기획/UX/사업/사용자 관점으로 같은 시나리오를 계속 반복 실행

## 1) Subagent 관점 정의

- `기획(Planning)`:
  - v1.1 핵심 퍼널(로그인 → 홈/MVP/탐색/프로필/팀)의 단계 누락 여부
  - 신규 유저 5분 시나리오의 끊김(빈 화면, dead button, 잘못된 라우트)
- `UI/UX`:
  - 탭 전환, 모달/시트 열림·닫힘, 포커스/접근성, 에러 메시지 전달력
  - 클릭 가능한 요소를 실제로 눌렀을 때 피드백이 있는지
- `사업(Biz)`:
  - 팔로우, 공유, 업로드, MVP 참여처럼 리텐션/확산 루프가 동작하는지
  - 공개 URL(`/p/{handle}`, `/t/{handle}`) 전환율 관점에서 막힘이 없는지
- `사용자(User)`:
  - "아무 생각 없이 탭/버튼 다 눌러도" 막히지 않는지
  - 에러가 나도 다음 행동으로 복귀 가능한지

## 2) 실행 커맨드

### 빠른 스모크 (비로그인 기준, 항상 재현 가능)

```bash
npm run test:e2e
```

보호 라우트(`/(home,mvp,discover,profile,team)`)는 비로그인 시 자동 `skip` 처리됨.

### 로그인 세션 저장 (1회 수동)

```bash
npm run test:e2e:auth:save
```

실행 후 열린 브라우저에서 카카오 로그인 완료 → `.auth/user.json` 저장.

### 전체 플로우 (로그인 상태 재사용)

```bash
npm run test:e2e:auth
```

### 전체 게이트 (타입 + 유닛 + e2e)

```bash
npm run test:qa
```

## 3) 시나리오 체크리스트 (클릭 기반)

- 인증:
  - `/login` 진입, 카카오 로그인 버튼 클릭, 인증 리다이렉트 확인
- 홈:
  - 베스트 캐러셀 카드 클릭
  - 피드 카드 응원/댓글/프로필 진입
- MVP:
  - 이번주/아카이브/명예의전당 탭 전환
  - 후보 카드/팔로우/투표 버튼 반응 확인
- 탐색:
  - 필터(전체/선수/팀/태그) 전환
  - 검색 오버레이 열기/닫기, 결과 클릭
- 프로필:
  - 요약/스킬/기록 탭 전환
  - 편집 시트 열기/저장/취소
  - 공유/추가 CTA 반응 확인
- 팀:
  - 팀 목록 진입 → 상세 → 멤버/앨범 전환
  - 팀 만들기/가입 시트 입력, 오류/성공 처리 확인
- 공개 페이지:
  - `/p/{handle}`, `/t/{handle}` 공유 시트
  - 멤버/선수 링크 전환

## 4) 운영 루프 권장

1. 배포 전: `npm run test:qa`
2. 스프린트 말: 로그인 상태로 전체 수동 클릭 리그레션 1회
3. 주간 리뷰: 실패 케이스를 `docs/sprints/SPRINT-18.md`에 누적
4. 기획 변경 시: `docs/SPEC.md`와 체크리스트 동기화

## 5) 증적(Artifacts)

- Playwright HTML 리포트: `playwright-report/`
- 실패 스냅샷/영상: `test-results/`
- 이슈 등록 시 각 증적 파일 경로를 티켓에 첨부
