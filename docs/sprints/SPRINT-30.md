# SPRINT-30: 클립 관리 UX 개선 (삭제/편집/관리 허브)

> Phase F (영상 UX 개편) | 의존: SPRINT-28 완료 (플레이어 개편)
> 예상: 3~4시간
> 우선순위: 3위

---

## 목표

클립 삭제/편집 접근성을 대폭 개선하고, 영상 관리 허브를 구축.
"어디서 지우지?"를 "바로 여기서"로 바꾼다.

---

## 현재 문제 (as-is)

- 삭제는 ClipPlayerSheet 내부에만 존재
- 프로필 클립 목록에서 직접 삭제/편집 불가
- 업로드 완료 화면에 삭제 버튼 없음
- 태그만 PATCH 가능, 메모 수정 불가
- long-press, swipe-to-delete 등 모바일 표준 패턴 없음

---

## 작업 목록

### 1. 클립 컨텍스트 메뉴 (long-press / ··· 버튼)

**파일**: 클립 썸네일 컴포넌트 (프로필 내)

- [ ] 클립 썸네일 우상단에 `···` 버튼 (44x44px 터치 영역)
- [ ] 또는 long-press(500ms) → 컨텍스트 메뉴
- [ ] 바텀시트 메뉴 항목:
  - 🏷️ 태그 편집
  - ✏️ 메모 수정
  - ⭐ 대표 영상 설정/해제
  - 🔗 공유
  - 🗑️ 삭제
- [ ] 삭제 선택 시 확인 다이얼로그 ("이 영상을 삭제하면 복구할 수 없습니다")

### 2. PATCH API 확장 (메모 수정)

**파일**: `src/app/api/clips/[id]/route.ts`

- [ ] PATCH body에 `memo` 필드 추가
- [ ] `skill_labels`, `custom_labels` 수정 지원
- [ ] 검증: memo 최대 200자, labels 최대 3+2개

### 3. 인라인 태그/메모 편집 UI

**파일**: 새 컴포넌트 `src/components/clip/ClipEditSheet.tsx`

- [ ] 바텀시트에서 태그 편집 (SkillLabelPicker 재활용)
- [ ] 메모 편집 (textarea)
- [ ] 저장/취소 버튼
- [ ] optimistic update + 실패 시 롤백

### 4. 업로드 완료 화면에 삭제 추가

**파일**: `src/components/upload/UploadComplete.tsx`

- [ ] "이 영상 삭제하기" 링크 버튼 추가 (하단, 빨간색 텍스트)
- [ ] 삭제 확인 다이얼로그 → 삭제 후 프로필로 이동

### 5. 프로필 편집 모드

**파일**: 프로필 페이지 클립 섹션

- [ ] "편집" 토글 버튼 (우상단)
- [ ] 편집 모드: 각 썸네일에 체크박스 표시
- [ ] 다중 선택 → 하단 액션바 ("N개 삭제" / "태그 일괄 변경")
- [ ] 편집 모드 해제 시 원래 상태로

### 6. 재편집 진입점 (중기)

**파일**: 컨텍스트 메뉴 + 새 로직

- [ ] "다시 편집" 메뉴 항목 추가
- [ ] 원본 영상(raw_url) DB에 보존 확인
- [ ] 원본이 있으면 → 위저드 Step 2(꾸미기)로 진입, 기존 설정 프리필
- [ ] 재렌더링 후 기존 클립 교체 (rendered_url 업데이트)

- [ ] 태그 + 메모 인라인 편집 가능
